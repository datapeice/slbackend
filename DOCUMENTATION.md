# Полная документация бэкенда StoryLegends

Этот файл содержит подробное описание всей внутренней логики бэкенда (Spring Boot), интеграций (Discord, RCON, Email), а также полный список API-эндпоинтов.

## Часть 1: Ключевой функционал и логика работы

Бэкенд StoryLegends — это не просто база данных, а центральный узел, который управляет сервером Minecraft, Discord-сервером и сайтом.

### 1. Авторизация, Регистрация и Безопасность

#### Регистрация аккаунта (`/api/auth/register`)
Процесс создания аккаунта строго защищен:
1. Проверяется, не закрыта ли регистрация в настройках админки.
2. Срабатывает базовая защита от ботов: проверяется валидность User-Agent и работает Rate Limiter (ограничение запросов с одного IP).
3. Проверяется токен **Google reCAPTCHA v3/v2**.
4. Проверяется уникальность логина, Email и Minecraft-ника.
5. Пароль хэшируется через `BCrypt`.
6. Бэкенд определяет геолокацию по IP (через `GeoIpService`) и записывает IP и User-Agent в профиль (для логов безопасности).
7. Генерируется уникальный токен подтверждения почты, после чего через `EmailService` отправляется HTML-письмо с кнопкой.
8. Аккаунт создается со статусом `emailVerified = false`.

#### Подтверждение почты (`/api/auth/verify-email`)
- Когда игрок переходит по ссылке из письма, фронтенд шлет токен на бэкенд.
- Бэкенд проверяет срок жизни токена. Если всё ок, ставит `emailVerified = true`.
- **Авто-вход:** Бэкенд сразу же генерирует и возвращает JWT-токен, чтобы игроку не пришлось вводить логин и пароль заново после клика по ссылке.

#### Вход в систему (`/api/auth/login`)
1. Проверка Rate Limit от брутфорс-атак.
2. Проверка логина и пароля.
3. **2FA (TOTP):** Если у пользователя включена двухфакторная аутентификация, бэкенд прерывает логин и требует 6-значный код из Google Authenticator. Только после проверки кода через `TotpService` вход одобряется.
4. При успешном входе генерируется JWT-токен, а также в базу данных записывается лог сессии (с какого IP и устройства зашел пользователь).

### 2. Система Заявок и Сброс Сезона

#### Подача заявки
Когда игрок подает заявку, бэкенд проверяет множество условий:
1. Открыт ли прием заявок (настройка в админке).
2. Аккаунт не забанен.
3. Почта подтверждена.
4. Discord привязан и пользователь **обязательно состоит на Discord-сервере** проекта (проверяется через JDA бота).
5. Пользователь не имеет флага `inSeason = true` (то есть не получал вердикт в текущем сезоне).
6. Проходится проверка Google reCAPTCHA.

**Что делает бэкенд при подаче заявки:**
- Создает заявку со статусом `PENDING`.
- Бот пишет игроку в личные сообщения Discord, что заявка отправлена.
- Бот пишет в специальный админский канал Discord о поступлении новой заявки.
- Записывает действие в Историю (Audit Logs).

#### Вынесение вердикта админом
Когда администратор принимает или отклоняет заявку, бэкенд выполняет целую цепочку автоматических действий.

**Если заявка ПРИНЯТА (`ACCEPTED`):**
- Статус игрока меняется: `isPlayer = true`, `inSeason = true`.
- **RCON:** Бэкенд подключается к Minecraft серверу по RCON и отправляет команду `whitelist add <ник>`.
- **Discord:** Бот автоматически выдает игроку роль `@SL` (или её аналог по ID) на Discord-сервере.
- **Discord:** Бот пишет игроку в ЛС поздравление с принятием и комментарий админа.
- **Email:** Отправляется красивое письмо о принятии (если эта функция включена в настройках сайта).

**Если заявка ОТКЛОНЕНА (`REJECTED`):**
- Статус игрока меняется: `isPlayer = false`, `inSeason = true` (блокирует повторную подачу до сброса сезона).
- **RCON:** Бэкенд отправляет команду `whitelist remove <ник>`, забирая проходку (на случай если она была).
- **Discord:** Бот забирает у игрока роль `@SL`.
- **Discord:** Бот пишет в ЛС об отказе с причиной.
- **Email:** Отправляется письмо об отказе.

#### Сброс сезона (Reset Season)
Кнопка сброса сезона (требует 2FA кода) выполняет один SQL-запрос: `UPDATE User u SET u.inSeason = false` для всех пользователей.
- **Она НЕ делает вайп!**
- Она не убирает статус `isPlayer` и не удаляет из вайтлиста.
- Суть: всем пользователям, включая тех, кому ранее отказали, снова дается право подать одну заявку (`inSeason` становится `false`).

### 2. Интеграции и Сервисы бэкенда

#### RCON Service (Управление Minecraft-сервером)
Бэкенд имеет прямое TCP RCON-подключение к серверу Minecraft. 
- Используется для выдачи/забора вайтлиста.
- Используется для запроса снимка античита (отправка команды `sl_anticheat request <ник>`).
- Может сбрасывать пароли на самом сервере Minecraft (если стоит плагин на авторизацию).

#### Discord Service (Бот JDA)
Бэкенд содержит внутри себя полноценного Discord бота (JDA), который стартует вместе с сервером.
- Проверяет наличие пользователя на сервере перед подачей заявки (кэширует участников).
- Выдает и забирает роли по ID, настроенным в `application.yml`.
- Рассылает личные сообщения (DMs) пользователям при смене статуса их аккаунта.
- Уведомляет админов о новых заявках в указанный канал.

#### Email Service (Почта)
Рассылает красивые HTML-письма через SMTP (обычно Gmail или Yandex).
- Коды подтверждения при регистрации.
- Восстановление пароля.
- Уведомления о статусе заявки (если включено в настройках).

#### Anticheat Service (Система логов системы)
Бэкенд принимает POST-запросы от клиентского мода SL-Anticheat.
- Сохраняет полный слепок системы игрока (MAC-адрес, Hardware ID, запущенные процессы, моды, активные окна).
- При получении списка модов от игрока, бэкенд проверяет их по своей базе **Известных модов (Known Mods)** и сразу размечает: "Этот мод безопасен (Доверенный)", "А этот чит (Подозрительный)". Это позволяет админам видеть раскрашенные списки в админке.

#### Audit Log Service (История действий)
Бэкенд логирует абсолютно все важные действия админов и модераторов в базу данных (таблица `audit_logs`). От выдачи бейджика до просмотра конкретного лога античита (`ANTICHEAT_VIEW_SNAPSHOT`). Ни одно действие админа не проходит мимо логов.

## Часть 3: API Эндпоинты

### Авторизация и Пользователи (`/api/auth` и `/api/users`)
- `POST /api/auth/register` - Регистрация аккаунта. Бэкенд шлет письмо с кодом.
- `POST /api/auth/verify-email` - Проверка кода из письма.
- `POST /api/auth/login` - Авторизация, возвращает JWT токен.
- `GET /api/users/me` - Получить свои данные профиля.
- `PATCH /api/users/me` - Изменить свой профиль (ник, био, аватар).
- `GET /api/users/me/discord/callback` - Эндпоинт для привязки Discord (OAuth2).

### Заявки (`/api/applications`)
- `POST /api/applications` - Подача заявки. Вызывает всю логику (DMs, проверки, логи).
- `GET /api/applications/my` - Получить свою текущую заявку и историю прошлых.

### Админ-Панель: Пользователи и Заявки (`/api/admin`)
- `GET /api/admin/stats` - Дашборд статистика (цифры на главной админки).
- `POST /api/admin/reset-season` - Сброс флага `inSeason` всем игрокам.
- `GET /api/admin/applications` - Список заявок для админов (с пагинацией).
- `PATCH /api/admin/applications/{id}/status` - Одобрить/отклонить заявку. Самый нагруженный метод (вызывает RCON, Email, Discord).
- `GET /api/admin/users` - Список всех юзеров.
- `POST /api/admin/users/{id}/ban` (и `unban`) - Выдать/снять блокировку сайта.
- `PATCH /api/admin/users/{id}` - Прямое редактирование пользователя (можно выдать проходку руками, без заявки).

### Админ-Панель: Значки, Настройки и Страницы
- `POST /api/admin/badges` - Создать новый значок (бейдж) профиля.
- `POST /api/admin/users/{userId}/badges/{badgeId}` - Выдать значок конкретному игроку.
- `GET`, `PATCH /api/admin/settings` - Управление глобальными настройками (включить/выключить прием заявок).
- `GET`, `POST`, `PUT`, `DELETE /api/admin/pages` - Управление кастомными страницами. Бэкенд конвертирует Markdown из БД в HTML для выдачи на фронтенд.

### Античит (`/api/admin/anticheat`)
- `POST /api/anticheat` - *ПУБЛИЧНЫЙ*. Сюда мод отправляет телеметрию игроков.
- `GET /api/admin/anticheat/snapshots` - Список слепков для админки.
- `GET /api/admin/anticheat/snapshots/{id}` - Получить все детали слепка (процессы/окна). При `log=true` оставляет след в Аудит Логах.
- `POST /api/admin/anticheat/request/{playerName}` - Пнуть сервер по RCON, чтобы он собрал логи игрока.
- `GET`, `POST`, `DELETE /api/admin/anticheat/known-mods` - База модов. Кнопки (галочка/крестик) на сайте бьют сюда `POST` запросом, чтобы разметить мод.

### Аудит Логи (`/api/admin/logs`)
- `GET /api/admin/logs` - Получить историю действий всех администраторов.
